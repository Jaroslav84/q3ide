## Quick Start: Minimal cbindgen.toml for a cdylib

For a Rust cdylib that exports C-ABI functions, here is a minimal working configuration:

```toml
# cbindgen.toml - Minimal configuration for a C dylib
language = "C"
include_guard = "Q3IDE_CAPTURE_H"
autogen_warning = "/* WARNING: This file is auto-generated by cbindgen. Do not modify manually. */"

[export]
include = []
exclude = []
```

## Complete Configuration Reference

### Top-Level Options

```toml
# The language to output bindings in.
# Possible values: "C", "C++", "Cython"
# Default: "C++"
language = "C"

# An optional string of text to output at the beginning of the generated file.
# Default: none
header = "/* Licensed under MIT */"

# An optional string of text to output at the end of the generated file.
# Default: none
trailer = "/* End of generated code */"

# An optional name to use as an #ifndef / #define include guard.
# Default: none (no include guard emitted)
include_guard = "MY_LIBRARY_H"

# An optional string to output between major sections as a warning against editing.
# Default: none
autogen_warning = "/* Warning, this file is autogenerated by cbindgen. Don't modify this manually. */"

# Whether to include a comment with the version of cbindgen used to generate the file.
# Default: false
include_version = false

# An optional namespace to output. Only applicable with language = "C++"
# Default: none
namespace = "my_lib"

# An optional list of namespaces. Only applicable with language = "C++"
# Default: none
namespaces = ["org", "mylib"]

# A list of sys headers to #include (with angle brackets).
# Default: none
sys_includes = ["stdint.h", "stdbool.h"]

# A list of headers to #include (with quotes).
# Default: none
includes = ["my_other_header.h"]

# Whether cbindgen should generate default #includes.
# Default: true (auto-generates includes for stdint.h, etc.)
no_includes = false

# A list of lines to add after the #include block.
# Default: none
after_includes = ""

# Whether to add a `using namespace` for any namespace.
# Only applies to C++ output.
# Default: false
using_namespaces = []

# The style to use for braces.
# Possible values: "SameLine", "NextLine"
# Default: "SameLine"
braces = "SameLine"

# The preferred line length (used as a hint for formatting).
# Default: 100
line_length = 100

# The amount of spaces in a tab.
# Default: 2
tab_width = 2

# Whether to generate documentation comments.
# Default: true
documentation = true

# Style of documentation comments.
# Possible values: "c", "c99", "c++", "cxx", "doxy", "auto"
# Default: "auto"
documentation_style = "auto"

# How much documentation to include.
# Possible values: "short", "full"
# Default: "full"
documentation_length = "full"

# Line ending style.
# Possible values: "LF", "CR", "CRLF", "Native"
# Default: "LF"
line_endings = "LF"

# How to sort items in the generated output.
# Possible values: "Name", "None"
# Default: "Name" (for functions and constants)
sort_by = "Name"

# Whether usize/isize should be converted to size_t/ptrdiff_t.
# Default: true
usize_is_size_t = true

# An optional prefix to add before every item name.
# Default: none
# item_prefix = "MY_LIB_"
```

### [defines] Section

Map Rust `cfg` attributes to C preprocessor `#define` checks:

```toml
[defines]
# Map Rust cfg attributes to C preprocessor defines
"target_os = macos" = "Q3IDE_MACOS"
"feature = debug" = "Q3IDE_DEBUG"
```

### [export] Section

Controls what gets exported:

```toml
[export]
# A list of additional items to always include in the generated bindings.
include = ["MyStruct", "MyEnum"]

# A list of items to exclude from the generated bindings.
exclude = ["InternalStruct"]

# A prefix to add before exported item names.
prefix = "Q3IDE"

# Types of items to include in the generated output.
# Default: all item types
item_types = ["constants", "globals", "enums", "structs", "unions", "typedefs", "opaque", "functions"]

# A rule to apply to rename every item.
# Possible values: "None", "GeckoCase", "LowerCase", "UpperCase",
#                  "PascalCase", "CamelCase", "SnakeCase",
#                  "ScreamingSnakeCase", "QualifiedScreamingSnakeCase"
rename = {}

# A list of strings which, if found as a substring of the path,
# will cause the item to not be emitted.
# exclude_by_attr = []

# A rule for how struct bodies should be generated.
# Possible values: "type", "tag", "both"
# Default: "both"
# body_style = "both"

# Whether to generate a helper that checks the size/alignment of types.
# mangle = {}
```

### [export.rename] Section

Rename specific items:

```toml
[export.rename]
"COption" = "Option"
"CResult" = "Result"
```

### [export.body] Section

Add custom body content to types:

```toml
[export.body]
"MyStruct" = """
    bool IsValid() const { return x != 0; }
"""
```

### [fn] Section

Function-specific settings:

```toml
[fn]
# How to rename function names.
rename_args = "None"

# An optional prefix to put before every function declaration.
# prefix = "Q3IDE_API"

# An optional postfix to put after every function declaration.
# postfix = "Q3IDE_CALL"

# How to format function arguments.
# Possible values: "horizontal", "vertical", "auto"
args = "auto"

# How to sort functions.
sort_by = "Name"

# Whether to generate Swift name annotations for functions.
# Only applicable for C output targeting Swift interop.
swift_name_macro = ""
```

### [const] Section

Constant-specific settings:

```toml
[const]
```

### [macro_expansion] Section

```toml
[macro_expansion]
# Whether to expand bitflags! macros.

```

### [parse] Section

Controls how cbindgen parses Rust source:

```toml
[parse]
# Whether to parse dependent crates.
parse_deps = false

# A whitelist of crates to parse.
include = []

# A blacklist of crates to skip.
exclude = []

# Whether to clean generated files before regeneration.
clean = false

# Extra Cargo.toml files to parse.
extra_bindings = []

[parse.expand]
# List of crates to expand macros for (using rustc).
crates = []

# Whether to use all features when expanding.
all_features = false

# Specific features to enable when expanding.
features = []

# Which profile to use for expansion.
# Possible values: "Debug", "Release"
profile = "Debug"
```

## Complete Example: q3ide-capture cbindgen.toml

This is a practical configuration for the q3ide-capture dylib project:

```toml
# cbindgen.toml for q3ide-capture
# Generates q3ide_capture.h from Rust cdylib

language = "C"
include_guard = "Q3IDE_CAPTURE_H"
autogen_warning = "/* WARNING: This file is auto-generated by cbindgen. Do not modify manually. */"
tab_width = 4
documentation = true
documentation_style = "c99"
style = "both"
usize_is_size_t = true
no_includes = false

# Add custom header
header = """
/*
 * q3ide-capture: macOS window capture via ScreenCaptureKit
 * C-ABI interface for Quake3e engine integration
 */
"""

[export]
prefix = "Q3IDE"

[export.rename]
"CaptureHandle" = "Q3IDECaptureHandle"
"FrameData" = "Q3IDEFrameData"
"CaptureError" = "Q3IDECaptureError"

[fn]
# No renaming needed since Rust functions are already q3ide_ prefixed
rename_args = "None"
args = "horizontal"

[struct]
rename_fields = "None"

[enum]
rename_variants = "ScreamingSnakeCase"

[parse]
parse_deps = false
```

## Cargo.toml for the cdylib

The corresponding `Cargo.toml` should specify the crate type:

```toml
[package]
name = "q3ide-capture"
version = "0.1.0"
edition = "2021"

[lib]
name = "q3ide_capture"
crate-type = ["cdylib"]

[dependencies]
crossbeam = "0.8"

[build-dependencies]
cbindgen = "0.29"
```

## Generating the Header

### CLI method

```bash
cbindgen --config cbindgen.toml --crate q3ide-capture --output q3ide_capture.h
```

### build.rs method

```rust
// build.rs
fn main() {
    let crate_dir = std::env::var("CARGO_MANIFEST_DIR").unwrap();

    let config = cbindgen::Config::from_file("cbindgen.toml")
        .expect("Unable to find cbindgen.toml configuration");

    cbindgen::generate_with_config(&crate_dir, config)
        .expect("Unable to generate C bindings")
        .write_to_file("q3ide_capture.h");
}
```
